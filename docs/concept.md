# コンセプト

## 一言で

「おもてなし」を「表無し」と読み替えた、地元民だけが知る裏スポットを巡る街歩きアプリ。

## 設計思想

### 投稿者と探索者を分けない

全員が「歩いて発見し、共有する人」。投稿する人 vs 見る人ではなく、全員が同じサイクルの中にいる。

### ゲーム的サイクル

```
歩く → スポットに近づく → GPS通知 → 写真を撮る → バッジ獲得
                                                    ↓
                          投稿主にリアクションが届く ← リアクション選択
                                                    ↓
                    「共有してよかった」→ また投稿したくなる
```

### 投稿の動機付け

課題: 裏スポットを投稿しても何もレスポンスがない。

解決: 訪れた人の「足あと」が投稿主に届く仕組み。

- 写真（匿名）が届く
- 定型リアクション（「美味しかった！」等）が届く
- 訪問数が見える

数字ではなく**人の反応**が報酬になる。

### バッジ獲得の条件

スポットの半径15m以内に接近 + 写真撮影 = バッジ獲得。

- GPSで接近を検知して通知
- 写真撮影が「実際に行った証明」になる
- 投稿の信頼性も担保される

## ターゲット

出先や家の周りで、知っている人の少ない穴場を巡りたい人。

## 差別化

| 既存サービス | 表無し |
|-------------|--------|
| Googleマップ: 誰でも見れる情報 | 地元民だけが知る裏スポット |
| 食べログ: 投稿してもレスポンスがない | 訪問者のリアクションが届く |
| ポケモンGO: 架空のスポット | 実在する裏スポット |
| Instagram: 自己表現が目的 | 「教えてよかった」が目的 |
