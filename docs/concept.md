# コンセプト

## 一言で

「おもてなし」を「表無し」と読み替えた、地元民だけが知る裏スポットを巡る街歩きアプリ。

## 設計思想

### 投稿者と探索者を分けない

全員が「歩いて発見し、共有する人」。投稿する人 vs 見る人ではなく、全員が同じサイクルの中にいる。

### ゲーム的サイクル

```
歩く → スポットに近づく → サムネイルが浮遊 → タップ → 写真を撮る → 足あと
                                                                      ↓
                                  投稿主にリアクションが届く ← リアクション選択
                                                                      ↓
                              「共有してよかった」→ また投稿したくなる
```

### 投稿の動機付け

課題: 裏スポットを投稿しても何もレスポンスがない。

解決: 訪れた人の「足あと」が投稿主に届く仕組み。

- 写真（匿名）が届く
- 定型リアクション（「美味しかった！」等）が届く
- 訪問数が見える

数字ではなく**人の反応**が報酬になる。

### 足あとの条件

ユーザーの現在地から半径20m以内にスポットがある時、地図上でサムネイルのデザインが変化（拡大+フローティングアニメーション）。そのサムネイルをタップして写真を撮ると足あとが残る。

- 20m圏内のサークルで接近を視覚的に伝える
- 写真撮影が「実際に行った証明」になる
- 投稿の信頼性も担保される
- 足あとを残すと、地図上のサムネイルにスタンプが付く

### 写真は全て正方形

アプリ内の写真は全て1:1の正方形で統一。

- 地図上のサムネイル（正方形の枠付き画像）
- スポット詳細のヘッダー
- カメラ撮影フレーム
- リアクション画面のプレビュー
- 投稿フォームの写真エリア

統一感を出し、どの画面でも同じ画像がそのまま使える。

## ターゲット

出先や家の周りで、知っている人の少ない穴場を巡りたい人。

## 差別化

| 既存サービス | 表無し |
|-------------|--------|
| Googleマップ: 誰でも見れる情報 | 地元民だけが知る裏スポット |
| 食べログ: 投稿してもレスポンスがない | 訪問者のリアクションが届く |
| ポケモンGO: 架空のスポット | 実在する裏スポット |
| Instagram: 自己表現が目的 | 「教えてよかった」が目的 |
