# 画面仕様

## 1. ログイン画面

### 表示要素
- アプリ名「表無し」+ サブタイトル
- メールアドレス入力
- パスワード入力
- ログインボタン

### 動作
- ログインボタン → メイン画面（地図）に遷移

### API
- なし（MVP: 認証後回し。ボタン押したら地図に飛ぶだけ）

---

## 2. メイン画面（地図）

### 表示要素
- 全画面の地図（MapLibre GL JS）
- スポットの正方形サムネイル画像（ピンの代わり）
- ユーザーの現在地を中心に半径20mのサークル表示
- サークル内のサムネイル → 拡大表示 + フローティングアニメーション
- サークル外のサムネイル → 通常サイズ
- 足あと済みのサムネイル → 半透明オーバーレイ + 足あとスタンプ
- 現在地: 青い丸 + 白枠 + パルスアニメーション
- フッターナビ: フローティングバー（角丸・影付き）
  - 左: 地図タブ
  - 中央: 投稿ボタン（丸い黒ボタン、枠内に収まる）
  - 右: マイページタブ

### 動作
- サムネイルタップ → スポット詳細シートが展開（上に少し隙間あり）
- フッター中央ボタン → 投稿画面（ピン設置）
- フッター「マイページ」→ マイページ

### API
- `GET /spots` → スポットの座標・名前・画像を取得

### 補足
- 地図の初期表示は川崎駅周辺（lat: 35.5308, lng: 139.7030）
- 現在地を青い点で表示（Geolocation API）

---

## 3. スポット詳細画面（シート展開）

### 表示要素
- シート形式（上部に少し隙間があり、背後の画面がチラ見え）
- 正方形のスポット写真（横幅いっぱい、シート上端の角丸に沿う）
- カテゴリタグ（pill型）
- スポット名
- 星評価 + 足あと数
- 説明文
- 投稿者情報（アイコン + 名前 + 称号）
- 下部ボタン（状態により変化）

### ボタンの状態

| 条件 | ボタンの状態 |
|------|------------|
| サークル内から遷移 | 有効（黒背景、「足あとを残す」） |
| サークル外から遷移 | 無効（グレー背景、「近づくと足あとを残せます」テキスト） |
| 足あと済み | ヘッダー画像に足あとスタンプが表示（ボタン非表示） |

### 動作
- 画像タップ → 地図に戻る（シートを閉じる）
- 「足あとを残す」ボタン → カメラ画面

### API
- `GET /spots/{id}` → スポット詳細取得

---

## 4. カメラ画面

### 表示要素
- カメラプレビュー（黒背景・正方形のフレーム枠 1:1）
- シャッターボタン（下部中央・白い丸）
- 閉じるボタン（左上・×）
- 「スポットの写真を撮影」テキスト

### 動作
- シャッターボタン → 写真撮影 → リアクション選択画面に遷移
- ×ボタン → スポット詳細に戻る

### API
- なし（撮影はフロントローカル処理）

### 補足
- 撮影は正方形（1:1）で統一
- カメラはブラウザのMediaDevices APIまたは`<input type="file" capture="environment">`で実装

---

## 5. リアクション選択画面

### 表示要素
- ヘッダー（← 戻る + 「リアクション」タイトル）
- 撮影した写真のプレビュー（正方形）
- リアクション選択肢（定型文・タップで選択）:
  - 静かでよかった！
  - 美味しかった！
  - 景色が良かった！
  - また来たい！
  - 穴場すぎる！
- 「確定する」ボタン
- 「📬 投稿主に匿名で届きます」テキスト

### 動作
- リアクションタップ → 選択状態（黒背景に切り替わる）
- 「確定する」→ API送信 → 足あと完了画面
- 戻るボタン → カメラ画面

### API
- `POST /spots/{id}/stamp`
  - body: `{ "user_id": "...", "latitude": 35.6895, "longitude": 139.6917, "photo_url": "...", "reaction": "美味しかった！" }`
  - スポットの座標から半径20m以内（`STAMP_ALLOWED_RADIUS_METERS`、現在地サークルと同じ半径）にいない場合は 400/422 エラー

---

## 6. 足あと完了画面

### 表示要素
- 正方形の撮影写真（中央に大きく表示）
- 足あとスタンプ（写真の上に踏みつけアニメーションで出現）
- 「足あとを残しました！」テキスト
- 「○○を訪問しました」テキスト
- 「地図に戻る」ボタン

### アニメーション
- 写真が少し潰れる（stamp-squish）
- 足あとアイコンが上から落ちてきてバウンド（stamp-drop）
- 最終的に写真の上に足あとマークが乗った状態で止まる

### 動作
- 「地図に戻る」→ メイン画面

### API
- なし

---

## 7. 裏スポット投稿画面

### 7-1. ピン設置画面

#### 表示要素
- 全画面地図
- 中央にピン（固定位置）
- 閉じるボタン（左上）
- 「ピンを設置」ラベル
- 現在地（青い丸）
- 「地図を動かしてピンの位置を決めてください」ヒント
- 「この場所に決定」ボタン

#### 動作
- 地図をドラッグしてピン位置を調整
- 「この場所に決定」→ 投稿フォーム画面

### 7-2. 投稿フォーム画面

#### 表示要素
- ヘッダー（← 戻る + 「スポット情報を入力」）
- 正方形の写真エリア（破線枠）
  - 「撮影する」ボタン → 投稿用カメラ画面
  - 「ライブラリ」ボタン → クロップ画面
- スポット名入力
- おすすめポイント入力（テキストエリア）
- 「投稿する」ボタン
- フォーム部分はスクロール可能

#### 動作
- 「撮影する」→ 投稿用カメラ画面
- 「ライブラリ」→ 端末の写真選択 → クロップ画面
- 「投稿する」→ API送信 → 投稿完了画面
- 戻るボタン → ピン設置画面

### 7-3. 投稿用カメラ画面

#### 表示要素
- カメラプレビュー（黒背景・正方形フレーム 1:1）
- シャッターボタン（下部中央・白い丸）
- 閉じるボタン（左上・×）
- 「スポットの写真を撮影」テキスト

#### 動作
- シャッターボタン → 撮影 → 投稿フォームに戻る（写真セット済み）
- ×ボタン → 投稿フォームに戻る

### 7-4. クロップ画面（正方形切り取り）

#### 表示要素
- 黒背景
- 選択した画像を中央に表示
- 正方形のクロップ枠（白い枠線）
  - 三分割グリッド線
  - 4隅にコーナーマーク（太い白線）
- 枠外は半透明黒オーバーレイ
- 閉じるボタン（左上・半透明丸 + ×）
- 「完了」ボタン（右上・白pill）
- 「ピンチで拡大・ドラッグで位置調整」ヒント（下部）

#### 動作
- ピンチで画像拡大縮小
- ドラッグで位置調整
- 「完了」→ 投稿フォームに戻る（クロップ済み写真セット）
- ×ボタン → 投稿フォームに戻る

### 7-5. 投稿完了画面

#### 表示要素
- 正方形の投稿写真（中央に大きく表示）
- ピンが上から落ちてくるアニメーション（stamp-drop）
- 「投稿しました！」テキスト
- 「裏スポットが地図に追加されました」テキスト
- 「地図に戻る」ボタン

#### 動作
- 「地図に戻る」→ メイン画面

### API
- `POST /spots`
  - body: `{ "name": "...", "description": "...", "photo_url": "...", "lat": ..., "lng": ... }`

---

## 8. マイページ

### 表示要素
- ユーザーアイコン + 名前
- 称号（マスターガイド等）
- 探索度プログレスバー（%表示）
- 統計: 投稿数 / 足あと数 / 訪問地数
- 投稿したスポット一覧
- フッターナビ

### 動作
- 投稿スポットタップ → スポット詳細画面
- フッターナビ「地図」→ メイン画面
- フッターナビ中央ボタン → 投稿画面

### API
- `GET /users/me`

---

## 画面遷移まとめ

```
ログイン → 地図（メイン）
              ├→ サムネイルタップ → スポット詳細シート → カメラ → リアクション → 足あと完了 → 地図
              ├→ フッター中央ボタン → ピン設置 → 投稿フォーム ─┬→ 撮影する → 投稿用カメラ → フォームに戻る
              │                                              └→ ライブラリ → クロップ → フォームに戻る
              │                                              → 投稿する → 投稿完了 → 地図
              └→ フッター「マイページ」→ マイページ → スポットタップ → スポット詳細シート
```

---

## デザイン方針

### 写真は全て正方形（1:1）
- 地図上のサムネイル
- スポット詳細シートのヘッダー画像
- カメラ撮影フレーム
- リアクション画面のプレビュー
- 投稿フォームの写真エリア
- 完了画面の写真表示

### フッターナビ
- フローティングバー（角丸20px + 影）
- 左右にタブ（地図・マイページ）、中央に丸い投稿ボタン
- 枠内に収まるデザイン（はみ出さない）

### スポット詳細はシート展開
- ハーフシートなし、タップで直接フル展開
- 上部に少し隙間（背後がチラ見え）
- 画像タップで閉じて地図に戻る
